Мои заметки для дизайнера

Адаптивность

  • Мобильные версии сайтов верстайте в размере 320px
  • Для элементов, которые тянутся по ширине, продумайте поведение (ограничение) на разных экранах
  • В неочевидных местах указывайте: элемент фиксирован в размерах или зависит от ширины/высоты экрана; какие минимальные/максимальные размеры?

Переполнение

  • Константин Константинопольский
  • В вебе есть переносы
  • Если мы нарисовали ряд карточек, и предполагается, что их может быть больше/меньше: как они себя будут вести?

Текст

  • Заведите текстовые стили и старайтесь не отступать от них.
  • Как должны выглядеть ссылки в тексте?
  • Шрифт в инпутах менее 16px

Цвета

  • Проверяйте контрастность.
  • Заведите палитру цветов и старайтесь не отступать от неё.
  • Для удобства создания ховеров и состояний можно использовать HSL.
  • Не используйте прозрачность, если в этом нет необходимости — пишите конечный цвет.
  • В какой цвет красить шапку браузера?

Интерактивность

  • Если нужно, продумайте состояния ховеров.
  • Если нужно сделать горизонтальную/вертикальную прокрутку элемента, решите прячем ли мы скроллбар, оформляем ли его по своему, или оставляем стандартный.

Картинки

  • Фавиконка для сайта — svg или png до 512х512. Можно отдельно для светлой и темной темы.
  • Обложка для шеринга в соцсетях 1280×640

Системы контроля версий

VCS — Version Control Systems

Придуманы, чтобы следить за историей изменений исходного кода и комфортно работать большой командой. Самая популярная — Git.

GitHub • VS Code • GitLens

Репозиторий

Repository — хранилище исходного кода с историей изменений

  • Локальный (Local) — размещенный на компьютере разработчика
  • Удаленный (Remote) — размещенный на GitHub

git init — создать (инициализировать) репозиторий

git clone ОТКУДА КУДА — скопировать (клонировать) репозиторий на компьютер

Коммит

Commit — подписанная пачка изменений исходного кода

git add ПАПКИ/ФАЙЛЫ — добавить измененённые файлы в следующий коммит (застейджить). Застейджить всё: git add --all

git commit -m "СООБЩЕНИЕ" — создать коммит (закоммитить)

git revert КОММИТ — отменить коммит (ревертнуть)

Работа с удаленным репозиторием

git push — отправить коммиты (запушить)

git pull — подтянуть коммиты (спуллить)

Continuous Integration/Continuous Delivery (CI/CD) — подход к разработке, при котором задачи сборки, публикации, тестирования продукта полностью или частично автоматизированы.

Ветки

Branch — параллельная история изменений

git checkout -b ВЕТКА — создать ветку

git switch ВЕТКА — переключиться на ветку

git merge ВЕТКА — влить (смёржить) ветку в текущую

Возможности веток (GitHub flow)

  • Параллельно работать над новой сложной функциональностью, привязывая ветку к задаче (карточка в Trello, тикет в менеджере задач, Issue на гитхабе)
  • Переиспользовать чужие коммиты
  • Выделить «продуктовую» ветку, и релизиться только при её изменении
  • Опубликовать тестовую версию сайта и проверить функциональность
  • Создать Pull Request (запрос на слияние веток) и провести Code Review (проверку чужого кода перед слиянием)
